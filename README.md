# Asterisk IPRN Dashboard

Web dashboard for managing multi-supplier Asterisk PBX — DID number ranges, IVR routing, call statistics, and real-time monitoring.

## Features

| Feature | Description |
|---------|-------------|
| **Dashboard** | Live Asterisk status, active calls, memory, uptime |
| **Call Statistics** | CDR-based stats — ASR, ACD, calls/min, recent calls |
| **Access Analyzer** | Test-number dial workflow with Mobile SIM or Asterisk mode and IVR/IPRN results |
| **Suppliers** | Named SIP providers with IP-auth (per-supplier PJSIP identify) |
| **Numbers** | Number inventory grouped by country/prefix with rates, range import |
| **DID Routes** | Route DIDs to IVR/ring groups/extensions, pick from inventory |
| **IVR Menus** | DTMF menu builder with audio files and timeout actions |
| **Ring Groups** | Agent extensions with ring timeout and voicemail fallback |
| **Config Preview** | View auto-generated extensions.conf and pjsip.conf |
| **One-Click Apply** | Generate configs, deploy to /etc/asterisk/, reload Asterisk |
| **Authentication** | Session-based login with configurable credentials |

## Quick Deploy on VPS

```bash
ssh root@167.172.170.88

git clone https://github.com/Kanonsarowar/Asterisk-Configuration-.git /tmp/asterisk-deploy
cd /tmp/asterisk-deploy

sudo bash deploy.sh
```

The script will:
1. Install Node.js 22 and Asterisk (if not present)
2. Backup existing Asterisk configs
3. Install dashboard to `/opt/asterisk-dashboard`
4. Prompt for dashboard username/password
5. Create systemd service (auto-start on boot)
6. Configure UFW firewall rules

Dashboard: **http://167.172.170.88:3000**

## Architecture

```
167.172.170.88
├── Asterisk PBX (:5060 UDP)
│   ├── pjsip.conf      ← auto-generated by dashboard
│   └── extensions.conf  ← auto-generated by dashboard
└── Dashboard (:3000 TCP)
    ├── server.js        ← Node.js HTTP server
    ├── data/db.json     ← all configuration data
    └── public/          ← web UI
```

**Call flow:**
```
Supplier IP → :5060 → Asterisk (IP-auth identify) → from-supplier-ip
→ DID routing → IVR menu → Ring group → Agent extension
```

## Configuration

### Default Login
- Username: `admin`
- Password: `admin123`

Change via environment variables:
```bash
sudo systemctl edit asterisk-dashboard
# Add:
[Service]
Environment=DASH_USER=yourusername
Environment=DASH_PASS=yoursecurepassword
# Save, then:
sudo systemctl restart asterisk-dashboard
```

## Backup

The most important file to backup is `data/db.json` — it contains all your suppliers, numbers, routes, IVR menus, and settings:

```bash
cp /opt/asterisk-dashboard/data/db.json ~/db-backup-$(date +%Y%m%d).json
```
