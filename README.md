# Gulf-Premium-Telecom — Asterisk IPRN Dashboard

**IPRN Business Console** — one web panel for multi-supplier Asterisk: **number inventory** (DIDs + routing), IVR audio, trunk, call stats, CDR, optional MySQL billing/inventory sync, and live monitoring.

**→ Full operator guide: [PROJECT_INSTRUCTIONS.md](./PROJECT_INSTRUCTIONS.md)** (source of truth, env vars, workflow, MySQL, optional legacy UI).

## Features

| Feature | Description |
|---------|-------------|
| **Dashboard** | Live Asterisk status, active calls, memory, uptime; optional IPRN/MySQL strip |
| **Call Statistics** | CDR-based stats — ASR, ACD, calls/min, recent calls |
| **CDR / Balance / SIP log** | History, balances, SIP invite log |
| **Suppliers** | Named SIP providers with IP-auth (per-supplier PJSIP identify) |
| **Number inventory** | DIDs grouped by country/prefix, rates, bulk import; **same list builds `did-routing` in Asterisk** |
| **IVR Audio** | Ten fixed IVR slots (1–10) with audio upload |
| **Trunk / DID test / Config preview** | Network settings, route tester, generated `.conf` preview |
| **Panel admins** | Extra dashboard users in `db.json` (env user always works) |
| **One-Click Apply** | Regenerate configs, copy to `/etc/asterisk/`, reload Asterisk |
| **Optional MySQL** | DID list in `numbers` table; sync `number_inventory`; billing APIs when enabled |
| **Authentication** | Session login; `DASH_USER` / `DASH_PASS`; optional auto-apply on login |

## Quick Deploy on VPS

```bash
ssh root@167.172.170.88

git clone https://github.com/Kanonsarowar/Asterisk-Configuration-.git /tmp/asterisk-deploy
cd /tmp/asterisk-deploy

sudo bash deploy.sh
```

The script will:
1. Install Node.js 22 and Asterisk (if not present)
2. Backup existing Asterisk configs
3. Install dashboard to `/opt/asterisk-dashboard` (on **redeploy**, existing `/opt/asterisk-dashboard/data/db.json` is **preserved** so the UI does not reset to first-install data)
4. Prompt for dashboard username/password
5. Create systemd service (auto-start on boot)
6. Configure UFW firewall rules

Dashboard: **http://167.172.170.88:3000**

### Update after `git pull` (why it looked “unchanged” before)

`deploy.sh` now runs **`npm ci`** (or **`npm install`**) under `/opt/asterisk-dashboard` so new code and dependencies (e.g. `mysql2`) actually load. Always **pull the latest repo** on the server, then redeploy:

```bash
ssh root@167.172.170.88
cd /tmp/asterisk-deploy && git pull
sudo bash /tmp/asterisk-deploy/deploy.sh
```

Then hard-refresh the browser (**Ctrl+Shift+R**). Config changes to Asterisk still require **Apply & Reload Asterisk** in the UI (or your own copy to `/etc/asterisk/`).

## Architecture

```
VPS (example)
├── Asterisk PBX (:5060 UDP)
│   ├── pjsip.conf, extensions.conf, acl.conf, rtp.conf, func_odbc.conf … ← generated by dashboard, copied to /etc/asterisk/
│   └── Dialplan: from-supplier-ip → did-routing → IVR (and/or ODBC path when enabled)
└── IPRN Business Console (:3000 TCP)
    ├── server.js
    ├── data/db.json     ← suppliers, IVR, trunk, globals, adminUsers, …
    └── public/          ← web UI
```

**Call flow (conceptual):**

```
Supplier IP → :5060 → Asterisk (IP-auth) → from-supplier-ip → did-routing
→ IVR context (or ODBC/PJSIP branch when IPRN ODBC routing is on)
```

Number inventory in the UI is what feeds **`did-routing`**; see [PROJECT_INSTRUCTIONS.md](./PROJECT_INSTRUCTIONS.md).

## Backup

The most important file to backup is `data/db.json` — it contains all your suppliers, numbers, routes, IVR menus, and settings:

```bash
cp /opt/asterisk-dashboard/data/db.json ~/db-backup-$(date +%Y%m%d).json
```
